Helmet For Node

2018-04-21

Node 前端开发

Hemlet

https://github.com/helmetjs/helmet
Helmet helps you secure your Express apps by setting various HTTP headers.
Helmet is a collection of 12 smaller middleware functions that set HTTP headers.

Content Security Policy

CSP模块是通过设置Content-Security-Policy header来保护网站不受到注入危险的Javascrpt,css…等。
用这个模块的话：需要设置一系列的白名单，js,css,images等等你需要保护的内容。
这类攻击最著名等就是XSS攻击－－－会将危险的Javascript注入进网页中，从而造成危险。

Expect-CT

Expect-CT 这个HTTP Header 会让浏览器去检查Certificate Transparency，来防止错误签发的网站证书的使用不被发现。

dnsPrefetchControl

通过设置X-DNS-Prefetch-Control header来取消浏览器的 DNS prefetching.
Prefetching 虽然可以让用户更加快速的连接，但是这会让用户浏览他们根本没有浏览过的网站，从而导致信息泄露。
PS: chrome 会自动把当前页面的所有带href的link的dns都prefetch一遍。 前端添加prefetch，兼容性不佳。

Frameguard

通过设置X-Frame-Options header 来降低Clickjacking的风险。
Clickjacking: 就是在真正的按钮上 覆盖了外链，导致用户点击了并不想点击的链接。

hidePoweredBy

通过设置X-Powered-By header，可以让黑客不知道你所用的技术是什么，甚至可以设置成错误的技术源，来误导不怀好意的用户。

hpkp

通过设置 Public-Key-Pins header来保护网站不备伪造。
通过伪造certificate authorities 可能可以伪造https的网站，来造成威胁。

hsts

通过设置 Strict-Transport-Security 来告诉浏览器只访问网站的https版本，拒绝访问http。
但是这个并不会强行跳转http－https,express-enforces-ssl这插件可能可以，没试过。

eNoOpen

X-Download-Options：noopen. 这个是用来防止IE8的bug，不让用户直接在网站上修改html。

noCache

这不是防御攻击，只是不让浏览器缓存你的js,css..等文件。以防当你更新了相应版本，浏览器缺读取的是旧文件。

noSniff

X-Content-Type-Options 保证浏览器去读取正确（安全）的文件类型，你传过去是什么Mimetype就是什么Mimetype。
不会去判断这个文件的类型，并执行相应操作。

referrerPolicy

Referrer-Policy 可以保证网站不知道用户是从哪个网站跳转过来。
如果不加上，example.com -> mdn.com. mdn.com就可以知道用户是从example.com点击跳转而来。

XSS Filter

XSS攻击， 最主要的目的就是可以控制你网站上的Javascript代码，从而做些什么。
X-XSS-Protection header 可以防御一种特定的通过浏览器url query产生的xss攻击。
设置了这个可以让浏览器不执行在url种javascript。
